## üöÄ Getting Started

### Prerequisites

Make sure you have these installed:

- **Node.js** v18.0.0 or higher
- **npm** v9.0.0 or higher (or **yarn** v1.22.0+)
- **Git** for version control

### Installation

1. **Clone the repository**

```bash
git clone https://github.com/jjr6639/thedevsite.git
cd thedevsite
```

2. **Install dependencies**

```bash
# Using npm
npm install

# Using yarn
yarn install
```

3. **Start development server**

```bash
# Using npm
npm run dev

# Using yarn
yarn dev
```

4. **Open in browser**

Navigate to `http://localhost:5173`

The development server includes:
- ‚ö° Hot Module Replacement (HMR)
- üîÑ Auto-refresh on file changes
- üìä Build performance metrics

### Available Scripts

| Command | Description |
|---------|-------------|
| `npm run dev` | Start development server with HMR |
| `npm run build` | Build optimized production bundle |
| `npm run preview` | Preview production build locally |
| `npm run lint` | Run ESLint code quality checks |

### Customizing Colors

#### Neon Green Theme

Change the primary color in `tailwind.config.js`:

```javascript
colors: {
  'neon-green': '#YOUR_COLOR',    // Primary
  'neon-dark': '#YOUR_DARK_COLOR', // Hover/Active
}
```

Update corresponding CSS variables in `index.css`:

```css
:root {
  --neon-green: #YOUR_COLOR;
  --neon-dark: #YOUR_DARK_COLOR;
}
```

#### Alternative Color Schemes

**Cyberpunk Purple:**
```css
--primary: #FF00FF;
--primary-dark: #CC00CC;
```

**Matrix Green:**
```css
--primary: #00FF00;
--primary-dark: #00CC00;
```

**Neon Blue:**
```css

```

## üåê Deployment

### Netlify (Current Deployment)

This site is deployed on **Netlify** with automatic deploys from GitHub.

#### Setup Netlify Deployment

1. **Push to GitHub**

```bash
git add .
git commit -m "Ready for deployment"
git push origin main
```

2. **Connect to Netlify**
   - Visit [Netlify](https://netlify.com)
   - Click "New site from Git"
   - Authorize GitHub
   - Select `thedevsite` repository

3. **Build Settings**

```
Build command: npm run build
Publish directory: dist
```

4. **Environment Variables** (if needed)

```
# Example for contact form
VITE_EMAILJS_SERVICE_ID=your_service_id
VITE_EMAILJS_TEMPLATE_ID=your_template_id
VITE_EMAILJS_PUBLIC_KEY=your_public_key
```

5. **Deploy**
   - Click "Deploy site"
   - Site will be live at `https://runyondeveloper.netlify.app`

#### Custom Domain

To add a custom domain:

1. Go to Site settings ‚Üí Domain management
2. Add custom domain
3. Configure DNS records
4. Enable HTTPS

### Alternative Platforms

#### Vercel

```bash
# Install Vercel CLI
npm i -g vercel

# Deploy
vercel

# Production deploy
vercel --prod
```

#### GitHub Pages

1. Install gh-pages:

```bash
npm install --save-dev gh-pages
```

2. Add to `package.json`:

```json
{
  "homepage": "https://jjr6639.github.io/thedevsite",
  "scripts": {
    "predeploy": "npm run build",
    "deploy": "gh-pages -d dist"
  }
}
```

3. Add to `vite.config.js`:

```javascript
export default defineConfig({
  base: '/thedevsite/',
  // ... other config
})
```

4. Deploy:

```bash
npm run deploy
```

## ‚ö° Performance

### Optimization Features

- **Code Splitting** - Automatic chunking for faster loads
- **Tree Shaking** - Eliminates unused code
- **Minification** - Compressed assets
- **Lazy Loading** - Images load on demand
- **Critical CSS** - Inlined for instant rendering

### Performance Metrics

Target Lighthouse scores:

| Metric | Score |
|--------|-------|
| Performance | 95+ |
| Accessibility | 100 |
| Best Practices | 95+ |
| SEO | 100 |

### Bundle Analysis

```bash
# Build and analyze
npm run build
npx vite-bundle-visualizer
```

### Performance Tips

1. **Images**
   - Use WebP format
   - Implement lazy loading
   - Compress before upload

2. **Fonts**
   - Use system fonts or subsetting
   - Implement font-display: swap

3. **JavaScript**
   - Minimize third-party scripts
   - Use dynamic imports for code splitting
